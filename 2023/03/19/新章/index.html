新章（此博客的创建过程）

Posted at 2023-03-19 tech  博客 tech 

使用工具

为了省钱和练手 GitHub，所以本次选用 Github Pages。又因为 Hexo 有中文页面，所以选用了它。

花费时间

从动手到最终建成，前后历时 4 天，共约 16 小时。

我看了很多经验分享，很多人在 7-10 小时左右。而我耗时这么长是因为部署时出现了一些 bug，还有部署后，主题页面显示有问题，而我找不到解决办法。随后尝试了新主题，但可能因为主文件被我改过，所以部署后还是很奇怪。最后决定删掉一切重新来过。当然，最后结果也并不是让我十分满意，但是我觉得目前已达到我最好水平。所以决定先这样，日后慢慢学着一步步修改。

教程

本次觉得比较好的教程，作者讲得比较详细。他应该也是新手，但是胜在够详细，英语阅读水平相对也比一些国内博主高一点。

使用主题

我一开始选了 clean 主题，但是出现 bug 之后，网上很难找到相关主题的问答。随后看到上面的教程，博主提到 volatis 的优点之一是有人维护，问答快，于是我就换了这个。我刚发布第一篇文章，想再修改页面设置，就出现 bug。网上找不到类似问答，我就去主题页面留言，第二天就收到了回复。追问也有人回答，对方非常友好。我有种“我终于有站友”“而不再孤单”的感觉！整个人是开心的，同时也更想快点完善配置，达到自己想要的效果。强推这个主题！

一些跳过的坑

部署时，branch 跟着一些帖子填了 master，但其实是 main。有些帖子将之改成 ph-page 之类，于我来说也是无用的。

遇到问题多方查找答案，chatGPT 此次就没解决我的问题，甚至不理解我的问题。有些网站也没有我遇到的问题，此时，就需要主动写帖发问了。

一些建议

先了解所有工具的基本框架，再仔细阅读。很多步骤可以跟着教程无脑跟，但是必要时刻还是得自己动脑。如果像我这样本来编程水平很低，那就更需要细心。

尽量找较多人使用的主题，这样遇到问题也容易寻求帮助。我第一次的主题，基本找不到类似的帖子，所以我才决定换一个。

相对来说，搭建这个博客并不需要很多知识，跟着教程一步一步走便是 。所以，耐心点，总会有所成。
